AI bbox accept flow — lightbox mockup

When a screenshot's label has ui_element_anchors from the AI, the lightbox renders each as a dashed indigo "ghost" rectangle over the screenshot. The right panel lists the same anchors with accept / reject / edit controls. Clicking a ghost OR an anchor row selects it (with resize handles); clicking the green tick promotes it into a real annotation row and the ghost turns solid green momentarily before settling as a normal annotation.

Implementation notes for the React side:
· Each ghost is positioned %-based off the bbox (matches how the existing area-annotation editor positions handles).
· Resize handles re-use the existing area-annotation resize logic. No new geometry math.
· "Accept" calls upsertAnnotation(...) with the current bbox + name + tagAsUiElement: true so it lands in both screenshot_annotations and the label's ui_element_anchors (the bbox-aware promote we just shipped).
· "Dismiss" removes the anchor from label.screen_analysis.ui_element_anchors on save — no annotation row created.
· "Undo" toggles state back to pending. Until the lightbox closes / saves, all state is local.
· Ghost layer is non-clickable while the area-annotation draw tool is active (otherwise drag-to-draw would collide).
