What's New panel + App-update toast

Two notification surfaces that work together. The toast prompts users to refresh stale JS; the panel reveals what changed after they do. Both can also be triggered independently — the ✨ icon in the header opens the panel any time, and the toast only fires when a new build is detected.

How the two connect

1. Build deploys build-id.json changes server-side
→
2. Toast appears Bottom-right, sticky. Click Refresh
→
3. Panel opens On first load of new version. ✨ in header for later

A new version is available

Refresh to get the latest fixes and features.

• Sticky — doesn't auto-dismiss. Stays until the user acts.

• Three actions: Refresh (primary), Later (snooze for ~30 min), × (dismiss until next build).

• Visual cue: indigo→green gradient stripe on the left edge so it's distinct from the existing toast (no left stripe).

• Polling: uses Page Visibility API — only polls /build-id.json when the tab is visible. 5 min interval.

• "Later" hides for 30 min then reappears if still stale. × hides for the current build only.
