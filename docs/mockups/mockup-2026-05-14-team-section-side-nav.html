CatalogueTeamSection — Side nav + clickable flows + bigger icons + toasts

Four feedback items rolled into one redesign of the Team section.

Feedback being addressed

  1. Side navigation for the 6 subtabs (Upload Analytics, Flows, Groups, Trash, Flags, Members) with Lucide icons.
  2. Click-to-filter: clicking a Flow or Group row activates the toolbar filter and switches back to the main catalogue, scoped to that flow/group.
  3. 2.5x bigger group icons in the Groups list (currently iconSize=14 → iconSize=36, rendered inside a tinted square so smaller icons don't look orphaned).
  4. Save messages become toasts — slide in, auto-dismiss after a couple of seconds instead of staying inline above the content.

What changes in code

  • Layout: .catalogue-team becomes a 2-col grid (200px nav + 1fr content) on viewports ≥ 768px. Below that, the nav collapses to a horizontal scrollable row of icon+label chips, or icon-only chips on the narrowest viewports.
  • Subtab buttons: existing 6 buttons keep their setSubTab wiring; just re-rendered as .team-nav__item with a Lucide icon + label + optional count badge (Flows/Groups/Trash have counts already available in state).
  • Click-to-filter: each row in Flows / Groups list gets an onClick that calls a new onSelectFlow(flow) / onSelectGroup(group) prop. Parent (Catalogue.tsx) wires it to setFilterFlow([flow]) + setActiveSection('catalogue'). Existing keyboard nav (Esc to back out) keeps working.
  • Group icons 2.5x: in the Groups list, render <CatalogueGroupLabel iconSize=36> inside a 36×36 tinted square wrapper. Falls back to a centered first letter when no uploaded icon. Other surfaces (toolbar chips, lightbox crumbs) keep the existing small size — only the Team Groups list changes.
  • Toasts: replace the inline .catalogue-team__group-note with the existing <Toast> component (already used in Catalogue.tsx). State stays as groupSaveMessage but gets cleared after 3s via a useEffect + setTimeout. Position: bottom-right of the page.
