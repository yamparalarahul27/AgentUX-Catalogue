CatalogueFamilyLightboxInlineEditor — Before / After v2

Iterated from v1 — Flow now uses a dropdown like Group, and Theme + Platform live on one row so the two toggles read as a group.

Before — current code

6 native <select> / text inputs stacked vertically. No grouping; every field gets a full row, vertical scroll grows quickly.

Group field: select + stacked text input (taller than siblings).

Flow: free-text only — no way to pick an existing flow, so consistency depends on the user typing the same string every time.

After — proposed v2

Group and Flow both use the same styled dropdown — pick existing or "+ Create new". One mental model.

Theme + Platform share one row as segmented controls. Together they're "what kind of screen is this?" — visually grouped.

Mobile OS / Web preset appears on its own row, only when Platform is selected. No empty field when not needed.

Code changes summary

  • Flow becomes a dropdown: add an existingFlows: string[] prop (parallel to existingGroups). The parent (CatalogueFamilyLightbox) already has access to the flows list — passes it through. Component swaps the <input> for a typeable Dropdown with "+ Create new flow".
  • Theme + Platform same row: wrap the two fields in a new .catalogue-list-inline-editor__row-2col div with grid-template-columns: 1fr 1fr. Mobile / narrow viewport: revert to single column via media query.
  • Theme / Platform / Mobile OS → segmented: small new <SegmentedControl> component (or inline buttons). Calls existing onThemeChange etc. with 'light' | 'dark' | null. No prop changes upstream.
  • Group dropdown: remove the second <input>. Reuse the Dropdown component from the toolbar. "+ Create new" lives in the dropdown menu.
  • Web preset: swap for Dropdown.
  • Behaviour preserved: all state shapes stay the same.
