Wiring

Six small pieces of code

What ships in the PR

1
Composer state: add tagAsUiElement: boolean + auto-derive normalizedLabel (trim, collapse whitespace, Title Case). Existing annotationDraftText stays as-is for the annotation row.
2
Toggle UI: new .lightbox-annotation-toggle row between the input and the action buttons in CatalogueFamilyLightbox.tsx.
3
Autocomplete: when toggle is on, render a .lightbox-annotation-suggestions list below the input. Source: pull from existing screen_analysis.ui_elements via the existing allUiElements prop already threaded through to the lightbox parent.
4
Near-match guard: tiny Levenshtein-distance check (≤2 chars) against existing UI Elements; render the amber warning row when a fuzzy match is found and the typed value isn't an exact existing one. Two CTAs: "Use existing" (replaces input value) and "Save as new anyway" (proceeds normally).
5
Save handler — two writes: (a) existing addAnnotation call; (b) if tagAsUiElement is true, fetch the screenshot's current metadata.label.screen_analysis.ui_elements, de-dupe-merge with the new normalized value, write back. Wrapped in a tiny helper promoteAnnotationToUiElement(screenshotId, label).
6
Badge on the annotation: render a small green "UI" pill on the annotation label when it's been promoted. Derived: any annotation whose label is also in the screenshot's ui_elements array. No new column needed.

Total: ~300 LOC across the composer JSX, a helper, and styles. No schema migration. No backend changes (Supabase RLS already lets the screenshot owner / admin patch their own labels).
