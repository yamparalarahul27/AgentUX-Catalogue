Catalogue

Your team's single source of truth for screenshots, feature tracking, design references, and competitive research — all in one place.

For Teams
Views & Organization
  • Four views: Grid, List, Gallery, and Folder (brand cards with carousel previews)
  • ThumbHash placeholders — instant blurry previews before full images load (28 bytes per image)
  • Upload and organize screenshots by group with full DB scope
  • Lightbox with annotations, comments, and inline editing
  • Bulk operations — rename, re-group, or delete multiple families at once
  • Global filters for group, platform, theme, flow, and search
  • Group appearance — custom icons, descriptions, and status badges per brand
Feature Log
  • Track every feature through Planned → Reference → Shipped lifecycle
  • Link existing screenshots as evidence for each status transition
  • Clickable feature cards with detail views showing shipped and reference evidence
  • Manual status override alongside title and description edits
Telegram Bot
  • Share screenshot images from your phone — they land in the "Social" group automatically
  • Paste X (Twitter) post URLs — they're added to the Videos section as embedded references
  • Allowlist-based access control so only your team can contribute
  • Duplicate detection — the bot tells you if a post is already in the catalogue
Archive & Compare
  • Side-by-side flow comparison across primary and vs groups
  • Read-only archive surfaces for Flow Builder and Projects
  • Full history preserved — nothing is deleted when you iterate
Videos & References
  • Curated video library for design references and competitive UX studies
  • Embed X (Twitter) posts with native rendering via Twitter Widgets API
  • Add references from the web UI or directly from Telegram
  • Comments per video for team discussion and annotation
Team & Analytics
  • Team upload analytics with per-member contribution tracking
  • Group management across the full database scope
  • Shared Supabase backend — all data syncs across Catalogue and Flow Builder

Workflow

Bulk Webapp-to-Figma Capture

What it does

Captures authenticated web apps into Figma at three breakpoints — automatically.

  • Logs into auth-gated products using a persistent browser session
  • Screenshots every state at 1512px (desktop), 720px (tablet), and 320px (mobile)
  • Imports frames into a single Figma file organized by user journey sections
  • Preserves previous captures in archive/legacy sections — nothing is deleted
  • Pauses for manual login when anti-bot challenges appear, then resumes

What you get

A ready-to-annotate Figma file with every screen in your product.

  • Figma sections: Core Journey, Extended Auth, Orders & History, Account & Wallet, Product Areas
  • Frames named with UJ-## / EX-## conventions across all three widths
  • A run log mapping each state to its capture ID and status
  • Old/incorrect frames moved to archive — full history preserved
  • Re-runnable from a single script + SOP with no tribal knowledge needed
Quick start 
URL=https://app.example.com \
PROFILE_DIR=/tmp/app-session \
PAUSE_FOR_LOGIN=1 \
KEEP_OPEN=1 \
node scripts/bulk_webapp_capture_template.cjs

Uses Playwright with a persistent Chrome profile. Pairs with the full SOP for team use.

FAQ

Common questions

Is AgentUX a framework?

No. It is a React and TypeScript library you install as a dev dependency. It works alongside your existing framework — Next.js or React Router.

Who is this for?

Anyone building apps with AI coding agents. Dev Mode is for developers inspecting routes. Catalogue is for teams managing screenshots, tracking feature lifecycle, collecting design references, and comparing flows across iterations.

What frameworks does Dev Mode support?

Next.js App Router, Next.js Pages Router, and React Router. Framework detection is automatic based on your project structure.

Can I add screenshots from my phone?

Yes. The Telegram bot lets you share screenshot images or X (Twitter) post URLs directly from your phone. Images go to the Catalogue and X links go to the Videos section — no desktop needed.

Is this analytics?

No. AgentUX is a development-time UX workspace, not production session replay or funnel analytics.
