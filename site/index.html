What you get with AgentUX

Dev Mode

Install in your app. See routes and flows in real time.

For Developers
What it does
  • Auto-detects Next.js App Router, Pages Router, and React Router
  • Tracks runtime route transitions as users navigate
  • Identifies dead ends, orphan routes, and hub pages
  • Exports current flow as Markdown + structured JSON for your AI agent
Quick start

  return (

    </>
  );
}

Flow Builder

Visual canvas to build and shape UX flows from screenshots.

For Designers & PMs
What it does
  • Drag-and-drop screenshots onto an infinite canvas
  • Auto-connect screens based on naming conventions
  • Compare screenshot versions side by side
  • Export the flow as a structured Markdown spec for AI
How naming works

Name files like 01-auth-login.png and Flow Builder auto-parses sequence, group, and screen name. Screens in the same group connect automatically.

Catalogue

Central screenshot library across all your projects.

For Teams
What it does
  • Upload and organize screenshots by project and group
  • Rename, re-group, replace, or delete from one place
  • Assign screenshots to flows — changes sync to Flow Builder automatically
  • Search and filter across all projects
How sync works

Catalogue and Flow Builder share the same database. Uploading, renaming, or assigning a screenshot in either place is reflected everywhere instantly.

FAQ

Common questions

Is AgentUX a framework?

No. It is a React and TypeScript library you install as a dev dependency. It works alongside your existing framework — Next.js or React Router.

Who is this for?

Anyone building apps with AI coding agents. Dev Mode is for developers inspecting routes. Flow Builder is for designers and PMs shaping the journey visually. Catalogue is for teams managing screenshots across projects.

What frameworks does Dev Mode support?

Next.js App Router, Next.js Pages Router, and React Router. Framework detection is automatic based on your project structure.

How do Flow Builder and Catalogue stay in sync?

Both read from the same database. Uploading, renaming, or assigning a screenshot in either tool is reflected in the other immediately on next load.

Is this analytics?

No. AgentUX is a development-time UX workspace, not production session replay or funnel analytics.
